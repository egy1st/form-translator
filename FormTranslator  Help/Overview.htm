Overview

Form Translator
Form Translator is the only program that gives you multi-language functionality without rewriting your code or redesigning the entire form. Developed by EgyFirst Software, it allows you to support Eastern languages and adapts your form design to accommodate foreign characters. Form Translator is perfect for programmers writing in multiple languages and trying to reach international markets.

Automatic translation
Form Translator works by taking your original form and translating the code into several languages. That means you only need to write one program, rather than recoding and redesigning with every version. Use it with Form Flipper to automatically rotate your form if the language calls for it. This way, you can spend more time polishing your project and less time slaving over repetitive code.

Simple code integration
The Form Translator function takes up only one line of code and appears only in the Form_Load event, so it won’t clutter your program. You don’t have to define it at every point or keep scrolling back up as you code, which saves you even more time.

Runs on COM-based applications
Form Translator works with any COM-based application, including Delphi, Borland C++, Visual C++, and Visual Basic. To use one application with multiple languages, just define the languages and fields on a standard table and copy it onto your project. You can also set the translator to match the programming language you use, and generate codes for foreign characters.

Easy installation and setup
You do not need to be an advanced programmer in order to install and fully use Form Translator. Its setup program automatically registers the DC_FormTranslator10.dll file on your system. To integrate it into your project, simply select 'add reference' from the project menu.  Click the 'browse' button to locate all of these  DLL

1-  your DC_FormTranslator10.dll file, which by default is located in c:\windows\system32.
2-  AxInterop.MSDataGridLib.dll  which by default is located in c:\windows\system32.
3-  Interop.MSDataGridLib.dll  which by default is located in c:\windows\system32.
4- adodb.dll  which by default is located in c:\windows\system32.
